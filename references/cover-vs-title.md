# 封面 vs 文章标题：硬规则 + 3 种模式

## 硬规则

**`cover_title` 不能等于 `title`（含 prefix）**。重复 = 业余 = 阅读量低。

两层各自承担不同任务：

- **封面图** —— 信息流缩略图、朋友圈分享卡 —— 0.5 秒抓眼，靠**钩子**
- **文章标题** —— 阅读流顶部、历史列表、搜索结果 —— 精确定题，靠**描述 + 系列归属**

两层重复同一句话 = 浪费了一个层。如果两层不一样，能传达**两份信息**——这是免费的杠杆。

主流的高级公众号（沈帅波、刘润、L先生、半佛仙人）基本上从不让封面文字 = 标题文字。

## 3 种模式 + 决策树

### 模式 A · 故事 + 论点 split（推荐默认）

**适合**：内容有"个人叙事 + 强论点"两个清晰维度。

- `title`：系列前缀 + 个人叙事
- `cover_kicker`：个人叙事的破题版（22px 小字）
- `cover_title`：强论点 / 钩子句（大字主标题）

**例**：
```yaml
title: "AI 输出格式 · 我已经很少做 PPT 了"
cover_kicker: "我已经很少做 PPT 了 ——"
cover_title: "HTML 被严重低估"
```

读者看到：
- 信息流缩略图：被"HTML 被严重低估"打动
- 文章页：被"我已经很少做 PPT 了"叙事吸引
- 没有重复，两份钩子

### 模式 B · 强论点 single

**适合**：标题较长或文章只有一个核心论点，没有故事维度。

- `title`：完整描述（系列前缀 + 全部信息）
- `cover_kicker`：留空
- `cover_title`：短小有力的关键词或论点

**例**：
```yaml
title: "AI 输出格式 · 我用 HTML 替代了 PPT"
# cover_kicker 不设
cover_title: "HTML 被严重低估"
```

### 模式 C · 完全分离

**适合**：内容有多个钩子角度，希望封面和标题分别吸引不同心境的读者。

- `title`：钩子组合 A（系列前缀 + 一组）
- `cover_kicker` + `cover_title`：钩子组合 B（完全不同的另一组）

**例**：
```yaml
title: "AI 输出格式 · HTML 被严重低估"
cover_kicker: "对外的事不再做 PPT ——"
cover_title: "HTML 才是 AI 时代的对外格式"
```

## 怎么选模式？

```
Q1: 标题/内容是否有"故事维度 + 论点维度"两个清晰的钩子？
  YES → 模式 A
  NO  → 进入 Q2

Q2: 标题是否较长（> 16 字）或较复杂（多个子句）？
  YES → 模式 B
  NO  → 进入 Q3

Q3: 是否能编出两组完全不重复但都吸引人的钩子组合？
  YES → 模式 C
  NO  → 默认模式 A
```

## "故事"与"论点"的判断标准

### 故事 / 叙事

第一人称、个人经验、情景描述。

例：
- "我已经很少做 PPT 了"
- "用 AI 一年之后"
- "今天我把 Word 删了"
- "我是怎么做到的"

### 论点 / 钩子

强宣言、反直觉、品类倒置、数字对比。

例：
- "HTML 被严重低估"
- "AI 比 Excel 强 10 倍"
- "PPT 是过时的产物"
- "不是 AI 不行，是你不会用"

## 检测重复的方法

发完 `publish_apple_style.sh` 之后，肉眼比对：

```
cover_title == title 子串 ?     → 重复，必须改
cover_title 与 title 有显著差异 ? → ✓
```

不允许的例子：
- `title: "AI 输出格式：HTML 被严重低估"` + `cover_title: "HTML 被严重低估"` ❌（cover ⊂ title）

允许的例子（partial overlap OK）：
- `title: "AI 输出格式 · 我已经很少做 PPT 了"` + `cover_kicker: "我已经很少做 PPT 了 ——"` + `cover_title: "HTML 被严重低估"` ✓
  - 共享短语 "我已经很少做 PPT 了" 是"锚点"
  - 但 cover 还有额外的 "HTML 被严重低估"，title 还有 "AI 输出格式" 前缀
  - 两层各自传递了不重叠的信息
