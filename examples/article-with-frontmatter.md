## 你大概不知道，AI 还能直接给你做 PPT

这一年我和 AI 协作下来，做对外的方案、汇报、宣传材料，已经很少打开 PowerPoint 了。

我现在的做法很简单：把要点写给 AI，告诉它用 HTML 输出。AI 直接把文件给我，双击，浏览器就帮我打开了一份排版规整、配色得体的页面。

整个过程不到 10 分钟，也没用任何设计工具。

## HTML 这个词别劝退你，整件事跟代码没关系

我知道 HTML 这个词有点劝退 —— 很多人一看就觉得"这是程序员的事，跟我没关系"。

不是的。

你完全不需要懂代码，也不需要去复制粘贴。现在主流的 AI 工具，基本都直接把 HTML 文件给你 ——

- 用 Claude 这种自带 Artifacts 的 AI，它写完之后，你点一下"下载"，一个 .html 文件就到电脑里了。
- 用 Cursor、Trae 这类 AI IDE，AI 直接把文件保存到你指定的文件夹，你都不用动手。
- 其他 AI 平台，基本也都有"另存为"或者"导出 HTML"的选项。

拿到文件之后，双击，浏览器就帮你打开。

整个过程，你不用知道 `<div>` 是什么、`class` 是什么。AI 全帮你写好了，你只负责双击。

## 这一年我摸出的一条线：对内用 Markdown，对外用 HTML

上一篇我讲了 Markdown —— AI 给你的那些 `*` 和 `#`，其实是它在用最舒服的语言整理结构。

这一篇要讲的 HTML，是另一件事。

我和 AI 配合下来，慢慢摸出一个清楚的分工：

| 场景 | 用什么 | 为什么 |
|---|---|---|
| 对内的事 | Markdown | 好维护、好搜索、可以放进文档系统 |
| 对外的事 | HTML | 是成品，能直接发出去 |

更直白一点：

- **Markdown** 让 AI 给你的，是"以后还要继续改的内容"
- **HTML** 让 AI 给你的，是"现在就能交出去的成品"

公司流程、SOP、培训笔记、会议纪要、内部周报 —— 这些都让 AI 用 Markdown 给我。
方案、汇报、宣传册、数据看板、对外介绍页 —— 这些都让 AI 用 HTML 给我。

这两个场景的本质区别是：**前者是给人改的，后者是给人看的。**

## 我每天拿 HTML 来干什么

下面是几件我反复在做、效果都很好的事。

### 1. 用 HTML 做 PPT 的替代

我现在对外的演讲、汇报、培训内容介绍，已经很少打开 PPT 了。

直接让 AI 用 HTML 做出来，比传统 PPT 快得多。一来不用一页一页拖文本框、调对齐；二来排版反而更好看，因为 HTML 天生就是为了"在屏幕上看"设计的。

里面想加图片可以加，纯文字也完全 hold 得住 —— 文字密度高的内容，用 HTML 反而比 PPT 看着舒服，因为它的排版逻辑天生就是为长内容服务的。

### 2. 用 HTML 做数据可视化

如果我手上有一份数据，要做柱状图、折线图、雷达图来分析，HTML 是我现在的首选。

让 AI 直接生成一个能跑的页面，里面就是图表。我不用打开 Excel 折腾图表样式，也不用截图再贴到 PPT 里。整个数据展示的过程是连贯的：数据进去，页面出来，发给同事看。

而且 HTML 做出来的图表是矢量的，放大看也不糊，比从 Excel 截图清楚太多。

### 3. 用 HTML 做对外的宣传手册和方案

公司有时候要做 A4 大小的宣传手册、产品介绍页、活动方案 —— 这些场景我都用 HTML。

让 AI 排版好之后，在浏览器里"打印"一下，选择"另存为 PDF"，就得到一份 A4 大小、排版规整的 PDF 文件。

这种文件发给客户、合作方，阅读体验比一份花哨的 PPT 好得多 —— 因为它本来就是按"被认真读"的方式排的版，不是按"被快速翻"设计的。

### 4. 用 HTML 做活动方案的封面页

给团队培训的时候，我演示过一份用 HTML 输出的活动方案。

团队当场"哇"了一声 —— 不是因为工具有多新，是因为他们突然意识到：原来 AI 给的东西，可以不是一段灰扑扑的字，可以是一个有标题、有配色、有结构、能直接发出去的页面。

很多人对 AI 的印象还停留在"它能写一段文字"这一层。当他们第一次看到 AI 直接给他们一个成品，反应是不一样的。

## 什么时候不要用 HTML

我也得说一句反面的 —— 不是所有事情都该用 HTML。

- **同事要在 Word 里和你协作的内容**：你给一个 HTML，别人没法直接改，反而把协作搞复杂了。
- **需要走流程、留痕的正式文件**：用公司规定的格式，不要为了好看自己换。
- **简单的两三行文字**：让 AI 用 HTML 做一封三行的邮件，是杀鸡用牛刀。

HTML 的价值在"对外、要展示、要好看、要一次交付"。这之外的场景，老老实实用回 Word/Excel/PPT。

## 你今天就能试一次

打开 Claude（或者其他支持下载文件的 AI），把下面这段贴进去：

```
请用单文件 HTML 做一份 [XX 主题] 的 [PPT/海报/方案/数据看板]，
所有 CSS 和 JavaScript 都写在一个 HTML 文件里，
风格现代、配色和谐，直接给我一个 .html 文件，我下载下来在浏览器里看效果。

具体内容：
[把你的内容粘贴进来]
```

AI 输出完之后，点一下"下载"，或者用"另存为"保存成 .html 文件。

然后双击这个文件，浏览器就会把成品打开给你看。

## 最后

我做 AI 培训这两年，看下来一个感受是：很多人只用了 AI 的十分之一。

原因不是 AI 不行，是没人告诉他们，AI 输出的不一定只是一段字。

HTML 是一扇门。推开之后，你会发现 AI 能做的事比你想象的多得多。
